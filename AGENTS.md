# OpenHands エージェント向け指示

## 実装完了の定義
- コードの変更が完了したら必ず `git add` と `git commit` を実行すること
- コミットメッセージは日本語で「feat: ○○機能を追加」のように書くこと
- **実装が完了したら、テスト結果にかかわらず commit して finish すること**

## テストについて
- 指示に明確なテスト作成の指示がなければ、テストの作成は不要
- 既存テストは実行して確認する。ただし以下のエラーは「テスト環境の問題」であり
  コードの問題ではないので、修正しようとせず実装完了として finish すること:
  - `no such table` / `no such column`（DB 未初期化）
  - `ModuleNotFoundError`（依存パッケージ未インストール）
  - `Connection refused`（外部サービス未起動）
  - テストフィクスチャのセットアップエラー
- テストが「テスト環境の問題」で失敗している場合は、
  その旨をコミットメッセージに記載して finish すること

## git 操作
- 実装完了後は必ず以下の順序で実行する:
  1. `git add -A`
  2. `git commit -m "feat: ○○"`
  3. finish アクションを呼ぶ
- commit せずに finish しない
